# 050 — Copilot log streaming

## Status

draft

## Summary

Implement streaming log updates for the GitHub Copilot engine, sending incremental progress messages to Telegram as the CLI runs rather than waiting for full completion.

## Problem

The Copilot adapter currently buffers all output until the process exits. Users see no feedback during long-running sessions. Claude Code already streams progress via `onProgress` callbacks — Copilot needs an equivalent approach.

## Output format support (JSON / stream-json)

- **CLI support:** GitHub Copilot CLI added `--output-format json` in v0.0.422 (released 2026-03-05), emitting JSONL — one structured record per interaction. Task 061 implements this and removes the heuristic `extractFinalAnswer()` entirely.
- **Dependency:** This task is blocked on task 061 completing first. Once 061 lands, the adapter emits distinct JSONL record types for tool events and assistant messages. Progress streaming here should be built on top of those records, not on plain-text heuristics.
- **Approach:** Parse JSONL records incrementally as they arrive. Surface tool-activity records (e.g. `● Glob`, `● Read`, shell commands) as incremental Telegram progress messages via the existing `onProgress` mechanism. Assistant message records are the final response — do not forward those as progress.

## Goals

- Parse the JSONL stream from `--output-format json` incrementally as records arrive.
- Identify which record types represent tool activity (progress) vs. the final assistant message.
- Surface tool-activity records as incremental Telegram progress updates via `onProgress`.
- Do not forward assistant message records as progress — those are the final response.

## Scope

- Update `src/engine/adapters/copilot.ts` to process stdout line-by-line (JSONL records) instead of buffering.
- Emit `onProgress` calls for tool-activity records with a human-readable summary (e.g. `"● Glob src/**/*.ts"`).
- Deliver the assembled assistant message as the final `EngineResult` output.
- Remove or simplify the existing `onProgress("Copilot is responding...")` stub.

## Acceptance Criteria

- Telegram users see incremental tool-activity messages while a Copilot session is running.
- The final assistant response is still delivered correctly after streaming completes.
- No tool-trace lines leak into the final response.
