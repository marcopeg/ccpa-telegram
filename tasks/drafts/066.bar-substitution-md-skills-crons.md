# 066 — Bar substitution in .md skills and crons

## Status

draft

## Summary

Add bar-style placeholder substitution (for example `|bot.firstName|`) to `.md` skills and `.md` cron prompts. This should enable simple template expansion using the same context maps already injected into prompts.

## Context

- SKILL.md prompt bodies are currently sent to the engine as-is, with no substitution.
- Cron `.md` prompt bodies are wrapped with context headers but do not substitute placeholders in the prompt text.
- Cron frontmatter already supports `${VAR}` substitution before YAML parsing.

## Scope

### In scope

- Add bar-style substitution for `.md` SKILL.md prompt bodies.
- Add bar-style substitution for `.md` cron prompt bodies at execution time.
- Use the same resolved context map used for context injection to resolve bar placeholders.
- Leave unresolved placeholders intact so missing keys are obvious.
- Keep existing `${VAR}` frontmatter substitution for cron `.md` files unchanged.

### Out of scope

- Any changes to `.mjs` skills or `.mjs` cron handlers.
- Changes to config-level `${}` / `@{}` semantics.

## Requirements

1. Define a shared bar-substitution helper (syntax `|key|`).
2. For skills: resolve bar placeholders just before calling the engine, using the message context map.
3. For cron `.md`: resolve bar placeholders at execution time using the cron context map.
4. Add tests/fixtures for both skills and cron `.md` prompt substitution, including unresolved placeholders.
5. Update documentation to explain bar substitution after implementation.

## Acceptance Criteria

- `|bot.firstName|` and other context keys are expanded inside SKILL.md prompt bodies.
- Cron `.md` prompt bodies resolve bar placeholders based on the execution context (including `cron.*`).
- Missing keys are left in the prompt unchanged (no silent empty-string substitutions).
- Existing `${VAR}` frontmatter substitution continues to work as-is.
- Test coverage exists for skill and cron prompt substitution.
