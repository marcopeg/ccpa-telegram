# 064 — CLI skill/command installer + updater

## Status

draft

## Summary

Ship a CLI-bundled catalog of skills and commands with an install flow that copies items into a project, plus an update flow that upgrades installed items by version while protecting user-modified files.

## Problem

Skills and commands are currently just files that must be copied manually into project folders. There is no standardized way to install or update them, no version tracking, and no protection against overwriting user edits. As the CLI ships updated skills/commands, projects need a safe, discoverable way to install and upgrade without losing local modifications.

## Goals

- Bundle a catalog of built-in skills and commands inside the CLI distribution.
- Add a project-level install command (enabled/disabled by config) that:
  - Prompts for installing a skill or a command.
  - Lists available items, taking already installed items into account.
  - Copies the selected item into the correct project folder (e.g., `.agents/skills/` or `.hal/commands/`).
- Add an update command that:
  - Compares installed items against the CLI catalog by version.
  - Updates only when a newer version exists.
  - Skips items with no version metadata.
- Protect user modifications:
  - Detect local changes (via stored hash/metadata).
  - Avoid overwriting modified files unless explicitly confirmed.

## Context

- Related to skill metadata/frontmatter work (task 051).
- Commands live under `.hal/commands/*.mjs`; skills under `.agents/skills/`.
- The CLI already owns the distribution of built-in assets, which can be used as the source of truth.

## Constraints

- Install and update flows must be opt-in and disable-able via config.
- Do not overwrite locally modified items without an explicit confirmation step.
- Version comparison must be deterministic; items without version metadata are not eligible for automatic updates.
- Keep the interaction simple (prompt for type, show list, confirm install/update).

## Acceptance Criteria

- A CLI command exists to install a skill or command from the built-in catalog into a project.
- A CLI command exists to update installed skills/commands when newer versions are available.
- Installed items carry version metadata, and update logic respects it.
- The system detects local modifications and will not overwrite them by default.
- Items without version metadata are ignored by the update command.
