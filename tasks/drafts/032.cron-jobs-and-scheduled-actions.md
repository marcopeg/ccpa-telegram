# 032 — Cron jobs and scheduled actions

## Status

draft

## Summary

Add cron-job support to schedule tasks in time, including one-off timers and recurring schedules, with persistent storage so jobs survive hot reload and process restarts.

## Requirements

- Support defining cron jobs via configuration.
- Support creating cron jobs via slash commands.
- Treat configuration-defined cron jobs as immutable from chat commands.
- When a user creates a cron job via command, persist it under that user’s folder.
- Persist all cron jobs to filesystem storage so they survive hot reload and process restarts.
- On startup/reload, rehydrate and re-schedule persisted jobs.
- Support both one-off jobs (timers/date-based) and repeated cron schedules.
- Support job actions that can:
  - execute a prompt,
  - execute a command,
  - execute a skill.

## Acceptance Criteria

- Config-based jobs load and schedule correctly at boot.
- Command-created jobs are saved to user-scoped files and restored after restart.
- One-off jobs run once and are marked/cleaned appropriately.
- Recurring jobs execute according to schedule.
- Hot reload does not lose scheduled jobs.
- The scheduler supports prompt/command/skill action targets.
