# 060 — Stale session empty-response recovery

**Status**: draft

## Description

When the Claude adapter returns a successful but empty response while a session is
active, detect the condition and prompt the user via Telegram with an explanatory
message and an inline button to reset the session.

This scenario occurs reliably when the Claude CLI is configured against a non-native
backend (e.g. OpenRouter via `ANTHROPIC_BASE_URL`) and the stored session ID is from
a previous native Claude API session. The CLI exits with code 0 but produces no
assistant content, leaving the user with a blank reply and no indication of what
went wrong.

## Goals

- Detect the "successful but empty" condition in the Claude adapter result path.
- Send the user a human-readable Telegram message explaining the likely cause (stale
  session) and suggesting a reset.
- Provide an inline Telegram button that, when tapped, clears the stored session ID
  for that user and confirms the reset.
- Optionally re-send the original user message automatically after the reset so the
  user does not have to repeat themselves.

## Detection Criteria

All three conditions must hold simultaneously:

1. `result.success === true` (no error exit code from Claude CLI).
2. `result.output` is empty or blank after trimming.
3. A session ID was active for this user at the time of the call
   (i.e. `sessionId` was non-null when `execute()` was invoked).

If all three are true, treat the result as a stale-session failure rather than a
normal empty response.

## Constraints and Expectations

- Detection is **Claude adapter-specific**; other adapters have different empty-output
  semantics and must not be affected.
- The inline button action must be scoped to the user + project (same as `/reset`
  session semantics) — it must not affect other users.
- Re-sending the original message after reset is **optional** (UX preference):
  include it if it can be done cleanly; skip if it adds complexity.
- The existing `/reset` and `/clean` commands must continue to work as before.
- If the detection fires, the stale-session event should be logged at `warn` level
  with the project slug, user ID, and the empty output.

## Acceptance Criteria

- Sending a message when the Claude CLI returns empty + success + active session
  produces a Telegram reply like:
  _"The session appears to be stale (no response received). Tap the button below to
  reset it and try again."_ with a **Reset session** inline button.
- Tapping the button clears the stored session ID for that user and sends a
  confirmation (e.g. _"Session reset. Please send your message again."_).
- If re-send is implemented, the original message is forwarded to the engine
  automatically after reset and the response is delivered normally.
- The condition is not triggered when output is empty for other reasons (engine
  error, `result.success === false`, or no active session).
- No regression in other adapters or in the normal Claude empty-response path (no
  active session).
