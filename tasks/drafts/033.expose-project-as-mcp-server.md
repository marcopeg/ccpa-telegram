# 033 — Expose project as MCP server

## Status

draft

## Summary

Investigate and define how a project can be exposed as an MCP server, including configurable tool definitions mapped to commands/prompts and optional semantic-search indexing backed by SQLite.

## Goals

- Evaluate architecture options to run a project as an MCP server endpoint.
- Allow project config to define MCP tools mapped to:
  - existing commands,
  - prompt templates/workflows.
- Explore support facilities such as automatic content indexing/tokenization for semantic search.
- Evaluate SQLite as storage for indexing metadata, embeddings references, and retrieval state.

## Scope

- Research protocol and runtime integration options for MCP in this codebase.
- Define configuration schema for MCP exposure and tool declarations.
- Define execution model for tool calls, permissions, and safety boundaries.
- Assess indexing pipeline options (document discovery, chunking, tokenization, refresh strategy).
- Compare storage design choices with SQLite as baseline.

## Deliverables

- Decision memo describing recommended MCP exposure approach.
- Proposed config contract for MCP tools (command-backed and prompt-backed).
- High-level technical design for semantic search indexing using SQLite.
- Risks, constraints, and phased implementation plan for refinement.

## Acceptance Criteria

- A clear recommendation exists for how to expose projects as MCP servers.
- Tool-definition format in config is specified with examples.
- SQLite-based indexing approach is assessed with trade-offs.
- Open questions are documented for refinement discussion.
