# 028 — JSON++ config: comments and tolerant parsing research

## Status

draft

## Summary

Research options to support comments in JSON config files, with potential tolerance for trailing commas introduced around commented sections.

## Scope

- Evaluate supported formats and parsers (e.g. JSONC, JSON5, custom preprocessing).
- Compare trade-offs: strictness, DX, safety, maintenance, ecosystem compatibility.
- Assess whether a custom parser layer that strips comments and normalizes edge cases is viable.
- Define migration and backward-compatibility strategy with current JSON config files.

## Deliverables

- A short decision memo with recommended approach.
- A risk list of parsing edge cases and failure modes.
- A proposed implementation plan for introducing JSON++ config support.
