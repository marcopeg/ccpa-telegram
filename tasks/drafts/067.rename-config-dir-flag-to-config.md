# 067 — Rename config-dir flag to config

## Status

draft

## Summary

Rename the CLI flag `--config-dir` to `--config` (and keep `--config-dir` as a deprecated alias) to simplify the interface, then update help text and docs accordingly.

## Problem

`--config-dir` is a bit verbose and harder to scan in examples. A shorter `--config` flag would be clearer while preserving the same behavior.

## Goals

- Add `--config` as the primary flag for the config directory.
- Keep `--config-dir` as a backward-compatible alias (with deprecation messaging).
- Update CLI help output, docs, and examples to prefer `--config`.
- Ensure error messages and prompts mention `--config`.

## Constraints

- Do not break existing scripts using `--config-dir`.
- Keep behavior identical (path resolution, defaults, validation).
- Document the alias and deprecation clearly.

## Acceptance Criteria

- `--config` works everywhere `--config-dir` did, with identical behavior.
- `--config-dir` still works and prints a deprecation notice (if applicable).
- Help text and docs consistently prefer `--config`.
- Existing tests pass; new coverage added if needed.
