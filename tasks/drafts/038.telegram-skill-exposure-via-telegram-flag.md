# 038 — Telegram skill exposure via telegram flag

## Status

draft

## Summary

When exposing skills to Telegram, include only skills explicitly marked with `telegram: true`.

## Requirements

- Change Telegram skill exposure filtering to require `telegram: true`.
- Update the `/help` placeholder filtering logic:
  - remove/replace current `public: true` gating,
  - use `telegram: true` as the Telegram-visibility criterion.
- Update example skills metadata in project examples to use the new flag consistently.
- Ensure skills without `telegram: true` are not shown/exposed in Telegram.

## Acceptance Criteria

- `/help` only lists skills with `telegram: true`.
- Telegram command/skill exposure paths use `telegram: true` consistently.
- Example skill definitions are updated from `public: true` to `telegram: true` where appropriate.
- Backward-compatibility behavior (if any) is explicitly documented in refinement/implementation notes.
