# 030 — Codex provider default model propagation

## Status

draft

## Summary

Support optional `providers.codex.defaultModel` (example: `gpt5.2`) and pass it to the Codex CLI `--model` argument when project-level model is not explicitly defined.

## Requirements

- Support optional config key: `providers.codex.defaultModel`.
- If `providers.codex.defaultModel` is set and a project using the Codex engine does not define its own model, propagate this value to CLI invocations.
- The propagated value must be explicitly passed via Codex CLI `--model` option.
- If a project model is explicitly set, it takes precedence over provider default.
- If `providers.codex.defaultModel` is not defined, preserve existing behavior.

## Acceptance Criteria

- Config parsing accepts and exposes `providers.codex.defaultModel` as optional.
- Codex CLI receives `--model <providers.codex.defaultModel>` for projects without explicit model.
- Project-level model continues to override provider default model.
- No behavior change for non-Codex engines.
