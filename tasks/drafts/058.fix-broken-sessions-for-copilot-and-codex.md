# 058 — Fix broken sessions for Copilot and Codex

**Status**: draft

## Description
Recent changes appear to have broken session behavior for the Copilot and Codex engines. As a result, conversations are no longer reliably “in-session” (e.g., context/continuity is lost or session state is not being reused as expected).

## Goals
- Reproduce and characterize the regression for both Copilot and Codex (what changed, when, and in which flows).
- Identify the correct source of truth for “session” in each engine integration (IDs, working dir state, CLI flags, persisted artifacts, etc.).
- Restore consistent session continuity per Telegram user and per project without cross-user leakage.

## Acceptance Criteria
- Copilot and Codex both maintain conversation continuity across multiple messages for the same Telegram user.
- Sessions are isolated across different Telegram users.
- A regression test / manual test checklist is documented so future changes don’t re-break session handling.
