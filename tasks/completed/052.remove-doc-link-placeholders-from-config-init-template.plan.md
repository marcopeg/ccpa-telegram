# Plan: 052 — Remove doc link placeholders from config init template

## Goal

Remove `{{HAL_DOCS_*}}` placeholders from the init template and from the CLI; use hardcoded doc URLs in both so the template is self-contained and the code no longer depends on doc URL constants. Behavior (output YAML and "Next steps" links) stays the same.

---

## Phase 1 — Template: hardcode doc URLs ✅

**File:** `src/init-template.yaml`

1. Replace the line `# {{HAL_DOCS_CONFIG}}` with a comment containing the current config docs URL:  
   `# https://github.com/marcopeg/hal/blob/main/docs/config/README.md`
2. Replace the line `# {{HAL_DOCS_TELEGRAM}}` under "How do I find my Telegram userID?" with:  
   `# https://github.com/marcopeg/hal/blob/main/docs/telegram/README.md`
3. Replace the line `# {{HAL_DOCS_TELEGRAM_CREATE_BOT}}` under "How do I create a Telegram bot?" with:  
   `# https://github.com/marcopeg/hal/blob/main/docs/telegram/README.md#creating-a-telegram-bot`
4. Replace the second `# {{HAL_DOCS_TELEGRAM}}` (env-file comment) with the same Telegram docs URL as in step 2.

**Check:** No remaining `{{HAL_DOCS` in the file; `{{ENGINE_NAME}}`, `{{ENGINE_MODEL}}`, `{{PROJECT_CWD}}` remain (still substituted in code). — Done; verified no `{{HAL_DOCS` in template.
