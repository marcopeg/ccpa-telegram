# 028 — JSONC config support — Development Plan

> Task 027 (YAML config support) was already completed. The multi-format detection layer existed in `src/config.ts`.

## Phase 1: Add dependency ✅

### Step 1.1 — Install `strip-json-comments`

- Ran `pnpm add strip-json-comments` — installed v5.0.3.
- ESM-only, compatible with `"type": "module"`. Ships its own TypeScript types.

## Phase 2: JSONC parser integration ✅

### Step 2.1 — Import and use in `parseConfigContent`

Added `import stripJsonComments from "strip-json-comments"` to `src/config.ts`.

Rather than a separate `parseJsonc` helper, integrated directly into the existing `parseConfigContent` function (which task 027 already created as the format dispatch). Added a `"jsonc"` branch that strips comments/trailing commas before `JSON.parse`.

### Step 2.2 — Register `.jsonc` in the format detection layer

- Extended `ConfigFormat` type: `"json" | "yaml"` → `"json" | "jsonc" | "yaml"`.
- Added `{ ext: ".jsonc", format: "jsonc" }` to `CONFIG_EXTENSIONS`.
- The existing `resolveConfigFile` exclusivity check automatically covers `.jsonc` — no additional code needed.

## Phase 3: Config watcher ✅

### Step 3.1 — Add `.jsonc` variants to `CONFIG_FILES` in `src/config-watcher.ts`

Added `"hal.config.jsonc"` and `"hal.config.local.jsonc"` to the watch list. Task 027 had already expanded it with YAML variants; `.jsonc` was added alongside.

## Phase 4: Config writer ✅

### Step 4.1 — Reading: automatic via `parseConfigContent`

`config-writer.ts` already calls `resolveConfigFile` + `parseConfigContent` (from task 027). Adding `"jsonc"` to the format detection and parser was sufficient — no writer-specific changes needed for reading.

### Step 4.2 — Writing: comments not preserved (by design)

`serializeConfig` falls through to `JSON.stringify` for both `"json"` and `"jsonc"` formats. Added a comment documenting this. The `/model` command (sole consumer) prefers writing to `hal.config.local.*`, so the commented base config stays untouched.

### Step 4.3 — File detection: automatic via `resolveConfigFile`

Already handled by the task 027 infrastructure. No hardcoded paths to update.

## Phase 5: CLI and logging updates ✅

### Step 5.1 — Startup log: already format-agnostic

Task 027 changed the local-detection check to `regex /hal\.config\.local\.\w+$/` which matches `.jsonc` automatically. No change needed.

### Step 5.2 — CLI help text

Updated `--cwd` description: `hal.config.{json,yaml}` → `hal.config.{json,jsonc,yaml}`.

### Step 5.3 — `init` command: kept as `.json`

No change. The init template has no comments — `.json` is appropriate.

## Phase 6: Convert example config ✅

### Step 6.1 — Created `examples/hal.config.jsonc`

Created alongside the existing `examples/hal.config.yaml` (task 027 had already replaced the original `.json` with YAML). Content is equivalent to the YAML example.

### Step 6.2 — Added representative comments

Demonstrates:
- Line comments (`// Global settings shared across all projects`)
- Inline comments (`// Supported: claude, copilot, ...`)
- Block comments with commented-out section (`/* Rate limiting defaults ... */`)
- Trailing commas after array items and object entries
