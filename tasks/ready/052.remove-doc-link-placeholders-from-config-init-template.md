# 052 — Remove doc link placeholders from config init template

**Status:** ready

## Summary

The `src/init-template.yaml` file contains placeholder tokens (`{{HAL_DOCS_CONFIG}}`, `{{HAL_DOCS_TELEGRAM}}`, `{{HAL_DOCS_TELEGRAM_CREATE_BOT}}`) for documentation links. These are currently substituted in `src/cli.ts` when generating the init config. Remove the placeholders from the template and the substitution logic from the code; replace with hardcoded doc URLs in both the template and the CLI so the template is self-contained and the code is simpler. The URLs can be fixed later in one place.

## Context

- The init template is used by `buildYamlInitConfig()` in `cli.ts` to generate a starter `hal.config.yaml` for new users. The same doc URLs are used in the template (via substitution) and in the "Next steps" console output after init (lines ~296, ~301).
- Goal: no doc-link placeholders; template and CLI both use hardcoded URL strings. This removes `HAL_DOCS_BASE`, `HAL_DOCS_CONFIG`, `HAL_DOCS_TELEGRAM`, `HAL_DOCS_TELEGRAM_CREATE_BOT` and all their uses from `cli.ts`. Leave `HAL_DOCS_URL` unchanged (used elsewhere for generic documentation link).

## Constraints

- Replace each placeholder with the **same** URL that is currently in `cli.ts` (so behavior is unchanged); author will fix URLs later.
- Keep surrounding comment context in the template (e.g. "Full config docs:", "How do I find my Telegram userID?", etc.) and only replace the placeholder line with a comment line containing the URL.

## Tasks

### Template (`src/init-template.yaml`)

- [ ] Replace `{{HAL_DOCS_CONFIG}}` line with a hardcoded comment containing the config docs URL (same as current `HAL_DOCS_CONFIG`).
- [ ] Replace `{{HAL_DOCS_TELEGRAM}}` in the `access.allowedUserIds` comment with the hardcoded Telegram docs URL.
- [ ] Replace `{{HAL_DOCS_TELEGRAM_CREATE_BOT}}` in the `telegram` section comment with the hardcoded "creating a bot" URL.
- [ ] Replace the second `{{HAL_DOCS_TELEGRAM}}` (env-file comment) with the same hardcoded Telegram docs URL.

### Code (`src/cli.ts`)

- [ ] Remove constants: `HAL_DOCS_BASE`, `HAL_DOCS_CONFIG`, `HAL_DOCS_TELEGRAM`, `HAL_DOCS_TELEGRAM_CREATE_BOT`.
- [ ] Remove the three `.replace(...)` calls in `buildYamlInitConfig()` that substitute `{{HAL_DOCS_CONFIG}}`, `{{HAL_DOCS_TELEGRAM}}`, `{{HAL_DOCS_TELEGRAM_CREATE_BOT}}`.
- [ ] In the init "Next steps" output (~lines 295–302), replace `HAL_DOCS_TELEGRAM_CREATE_BOT` and `HAL_DOCS_TELEGRAM` with the same URLs as string literals so the printed messages are unchanged.

## Acceptance criteria

- [ ] `src/init-template.yaml` contains no `{{HAL_DOCS_*}}` tokens; each former placeholder line is a comment line with the corresponding URL.
- [ ] `src/cli.ts` does not define or use `HAL_DOCS_BASE`, `HAL_DOCS_CONFIG`, `HAL_DOCS_TELEGRAM`, or `HAL_DOCS_TELEGRAM_CREATE_BOT`; init still produces the same YAML and the same "Next steps" doc links.
- [ ] `pnpm run build` passes; init flow (template output and Next steps) is unchanged apart from URLs now living in template + string literals in CLI.
